AGI Analytics Dashboard & Demographics Merge Tool

Last updated: May 2026  |  Prepared by Drew Dahlberg

Purpose of this document: This manual covers everything your team needs to understand, operate, and maintain the Senior Housing Bot desktop application and the Demographics Merge website after the original developer's departure.

Part 1 — Senior Housing Bot

What Is It?

The AGI Analytics Dashboard (also called the Senior Housing Bot) is a Windows desktop application built in Python. It has one core job: automatically pull data on senior living communities from 8 major operators' websites, track changes over time, display everything on an interactive map, and help identify market opportunities using US Census demographic data.

Think of it as a live database of your competitors — every time you run a scrape, it captures what communities they operate, where they are, and flags anything new or removed since the last run.

What Does It Actually Do?

  • Scrapes 8 senior living operators and collects community names, addresses, phone numbers, URLs, and amenities
  • Geocodes every address (converts street addresses into GPS coordinates) so they appear correctly on the map
  • Tracks changes over time — when a new community opens or closes, it's flagged automatically
  • Shows everything on an interactive Mapbox map with color-coded markers per operator
  • Ranks markets by opportunity score using Census demographic data (age 75+, income, home values)
  • Merges demographic reports (1-mile, 3-mile, 5-mile radius) into a single formatted Excel workbook
  • Manages Offering Memorandums (OMs) — upload PDFs and pin them to the map

How Scraping Works

The Process (Step by Step)

Click "Scrape All Operators" — the app launches 8 background threads simultaneously, one per operator.
Each scraper fetches data — using one of three methods: direct API call, HTML parsing, or a headless Chrome browser (Selenium) for JavaScript-heavy sites.
Missing coordinates are geocoded — any community without GPS coordinates is automatically looked up via the US Census Geocoder (batch of up to 1,000 at once), with OpenStreetMap as a fallback.
Data is saved to Excel — each operator gets its own all_communities.xlsx file.
Changes are tracked — the app compares the new data to the previous run. New communities are marked New, existing ones get a checkmark ✓, and removed ones are marked Removed. This history is saved in community_tracking.xlsx.
Map refreshes — all communities appear as colored markers on the Mapbox map.

Change Tracking

Each operator has a community_tracking.xlsx file. Each column is a date a scrape was run. Each row is a community. The cell value tells you the status on that date:

ValueMeaning
NewFirst time this community appeared in the data
✓Community was present and active on this date
RemovedCommunity was in previous data but not found this time

Application Tabs

Map Tab

The main tab. Shows all scraped communities as colored dots on a Mapbox map. Each operator has a unique color shown in the legend. You can toggle operators on/off, zoom to a specific state, county, or ZIP code, and click any marker to see the community's full details.

Demographics Tab

Merges three Esri demographic radius reports (1-mile, 3-mile, 5-mile) into a single formatted Full Service workbook. Upload the three files, optionally enter an address, and click Merge. The output Excel file is saved to the senior_housing_data/demographics/ folder. This is the same functionality now available on the website.

Market Rankings Tab

Ranks counties, ZIP codes, or minor civil divisions by an opportunity score calculated from US Census data. Select a state, choose geography type, and click Fetch & Rank. The score is a weighted combination of:

  • Population age 75+ × % households earning $50k+
  • Population age 45–64 × % households earning $100k+
  • Median home value (normalized)

Results are cached in a local SQLite database (geo_stats.db) for 30 days so you don't have to re-fetch from Census every time.

Requires a Census API key. The key field in config.json is currently empty. Sign up free at api.census.gov/data/key_signup.html and paste the key into the app's settings or directly into config.json.

OM Upload Tab

Upload Offering Memorandum PDFs. Enter the property address, unit count, and notes. The app geocodes the address using Mapbox and pins the OM on the map. Records are stored in om_records.json and PDFs are stored in senior_housing_data/om_pdfs/.

Part 2 — Demographics Merge Website

What Is It?

A simple web tool that does exactly what the Demographics tab in the Senior Housing Bot does — but accessible from any browser, on any computer, without installing anything. Users upload their three radius demographic reports, click a button, and download the merged Full Service Excel file instantly.

Website URL: https://demographics-web-production.up.railway.app (or your custom domain once connected)

Password: AGI

How to Use the Website

Go to the website URL and enter the password AGI
Choose your upload method:
  • Upload 3 Files tab — upload each radius file individually (1-mile, 3-mile, 5-mile)
  • Upload ZIP tab — if you have all three files in a ZIP, upload that instead. The app detects which file is which automatically based on the filename.
Optionally enter the property address — this gets written into cell A1 of the Summary tab in the output. If left blank, it's auto-detected from the filename.
Click Merge & Download — the merged Excel file downloads to your computer automatically.

File Naming Convention

For automatic address detection, name your files like this:

Executive Dashboard - 201 Clintonville Rd - 1 mi.xlsx
Executive Dashboard - 201 Clintonville Rd - 3 mi.xlsx
Executive Dashboard - 201 Clintonville Rd - 5 mi.xlsx

The app reads the address between the dashes and writes it to the output automatically.

What the Output Contains

The output is a copy of the Full Service Demographic Template with data from all three radius reports merged in. Each radius has its own set of tabs:

All formatting, merged cells, column widths, and row heights from the source files are preserved in the output.

How It Works (Technical)

The website is a Python web application. Here is exactly what happens when you click Merge:

Your browser sends the uploaded files to the server (Flask web framework handles this)
The server saves them to a temporary folder
The DemographicsMerger class (in merger.py) copies the bundled template, then reads each radius file using openpyxl and copies the sheet data, formatting, merged cells, and column widths into the correct destination tabs
The merged file is sent back to your browser as a download
The temporary files are deleted — nothing is stored on the server

Hosting & Deployment

Where Is It Hosted?

The website runs on Railway (railway.app). Railway is a cloud hosting platform that runs the Python/Flask app 24/7. The code lives on GitHub and Railway automatically redeploys whenever the code is updated.

How Auto-Deployment Works

Railway is connected to the GitHub repository DREW2212/demographics-web. Every time code is pushed to GitHub, Railway detects it and rebuilds the app automatically — usually takes about 2 minutes. No manual steps needed.

How to Update the Website

To make any change (new feature, password update, template change):

Edit the relevant file locally in the demographics-web folder on the Desktop
Open a terminal in that folder and run:
git add .
git commit -m "describe the change"
git push
Railway picks up the push and redeploys automatically

How to Change the Password

Open templates/index.html and find this line:

const PASSWORD = 'AGI';

Change AGI to the new password, then push to GitHub.

How to Update the Template

Replace the file at template/Full_Service_Demographic_Template.xlsx with the new version. Make sure the sheet names inside it still match the destination tab names in the Sheet Mapping table above. Then push to GitHub.
